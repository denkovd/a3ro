## Phases

### P·06 — Regime Shift Finder (new · Darius-Dale-style macro regime)
**Backend SHIPPED 2026-07-11** (the shared FRED macro layer — see scores-plan #5): keyless `fredMacro.ts` (fredgraph CSV, no api key — `FRED_API_KEY` was empty; T10Y2Y live-probed fresh to 2026-07-10), pure `macro/engine.ts` (`computeMacroRegime` GRID + `computeMacroPressure`), `macro_snapshots` table (`009_macro.sql`), `runMacroCycle` wired into the daily cron (isolated), and `/api/oil/macro`. Fixture-tested. **UI SHIPPED 2026-07-11** — P·06 module card (`RegimeShiftFinder.tsx`: GRID quadrant dial + growth/inflation axis reads + a Macro Override pressure/divergence chip) and full page `/Projects/Regime-Shift` (quadrant panel + axis cards + Macro Override pressure legs), registered in the homepage module grid; `macroData.ts` hook on `/api/oil/macro`. (Optional later: also embed the Macro Override amber chip inside the oil-tracker rail — held to avoid an unverifiable edit to the 2.5k-line OilTrackerCore.)

A **genuinely new module**, not a rename of the two Bull Market Finder variants. Where P·04/P·05 (Bull Market Finder 1/2) answer *what is trending right now* from bottom-up price flips, P·06 answers *where are we in the growth/inflation cycle*. Build it as a **GRID-style growth × inflation regime**: two axes measured on a rate-of-change basis (accelerating / decelerating) → four quadrants — **Goldilocks** (growth↑ / inflation↓), **Reflation** (growth↑ / inflation↑), **Inflation/Stagflation** (growth↓ / inflation↑), **Deflation** (growth↓ / inflation↓) — with the current quadrant, its trajectory, and what it historically favors. Gets its own route/API/table (e.g. `/Projects/Regime-Shift`, `/api/regime-macro`, a `macro_regime` snapshot table) so it sits alongside the two Bull Market Finder variants rather than on top of them. No universe overlap — it is macro-economic data, not a price-trend list.

Data is free-tier **FRED** (probe every series id live before speccing, per build discipline): growth proxy `INDPRO` (industrial production, + payrolls/ISM if surfaced on FRED), inflation `CPIAUCSL`/PCE or market-based `T10YIE` / 5y5y breakevens for a forward read, context `T10Y2Y` (curve), `BAMLH0A0HYM2` (HY OAS), broad dollar `DTWEXBGS`. Method: YoY + momentum (2nd-derivative) per axis → quadrant classification + transition history, mirroring the Regime/Bull precedent (pure engine → snapshot table → read-only route → hook → card).

**Convergence — build the macro layer once.** This is the same FRED macro layer as scores-plan.md's **Macro Override → Macro pressure** half (sequencing #5). Build a single `fredMacro.ts` sibling adapter + engine and surface it in both P·06 (the regime card) and Macro Override (the score chip) — do NOT build two macro layers. This is the reason to plan it now rather than duplicate later.
